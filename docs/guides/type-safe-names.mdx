mujoco-react uses TypeScript declaration merging for model resource names. The recommended workflow is the same shape as TanStack Router route generation: a Vite plugin scans your source files and writes a checked-in generated type artifact.

## Vite Plugin

```ts
// vite.config.ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import { mujocoReact } from "mujoco-react/vite";

export default defineConfig({
  plugins: [
    react(),
    mujocoReact({
      models: {
        franka: "models/panda/scene.xml",
        spot: "models/spot/scene.xml",
      },
    }),
  ],
});
```

The plugin writes `src/mujoco-register.gen.d.ts` during dev and build:

```ts
// Auto-generated by mujoco-react. Do not edit.

import "mujoco-react";

declare module "mujoco-react" {
  interface Register {
    robots: {
      franka: {
        actuators: "joint1" | "joint2" | "gripper";
        sensors: "imu_gyro";
        bodies: "link0" | "hand";
        joints: "joint1" | "joint2";
        sites: "tcp";
        geoms: "floor";
        keyframes: "home";
      };
      spot: {
        actuators: "fl_hx" | "fl_hy" | "fl_kn";
        sensors: never;
        bodies: "body" | "fl_hip";
        joints: "fl_hx" | "fl_hy" | "fl_kn";
        sites: never;
        geoms: "floor";
        keyframes: "home";
      };
    };
    actuators: "joint1" | "joint2" | "gripper" | "fl_hx" | "fl_hy" | "fl_kn";
    sensors: "imu_gyro";
    bodies: "link0" | "hand" | "body" | "fl_hip";
    joints: "joint1" | "joint2" | "fl_hx" | "fl_hy" | "fl_kn";
    sites: "tcp";
    geoms: "floor";
    keyframes: "home";
  }
}
```

Commit this generated file. It gives autocomplete and type checking without adding runtime cost.

```tsx
useCtrl("gripper");
useSensor("imu_gyro");
useIkController({ siteName: "tcp" });
api.applyForce("hand", force);
```

## Per-Robot Helpers

The global unions keep existing hooks ergonomic. For reusable robot-specific controllers, use the per-robot helper types:

```tsx
import { useCtrl, useIkController } from "mujoco-react";
import type { RobotActuators, RobotSites } from "mujoco-react";

const gripperName = "gripper" satisfies RobotActuators<"franka">;
const tcpSite = "tcp" satisfies RobotSites<"franka">;

function FrankaController() {
  const gripper = useCtrl(gripperName);
  const ik = useIkController({ siteName: tcpSite });

  return null;
}
```

This keeps a Franka controller from accidentally depending on a Spot-only actuator name even when both robots are included in the same app.

## Multiple Models

```ts
mujocoReact({
  models: {
    franka: "models/panda/scene.xml",
    spot: "models/spot/scene.xml",
  },
  generatedRegister: "src/mujoco-register.gen.d.ts",
});
```

Names from all configured models are merged into the global `Register` unions, while `Register.robots` keeps the per-robot resource sets available for scoped helper types.

## CLI Fallback

Use the CLI for non-Vite projects or one-off generation:

```bash
npx mujoco-react codegen franka=models/panda/scene.xml spot=models/spot/scene.xml
```

Legacy alias:

```bash
npx mujoco-react-codegen franka=models/panda/scene.xml
```

## Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `models` | `string \| string[] \| Record<string, string>` | required | Entry MJCF/URDF files to scan. Use a record for stable per-robot type names |
| `generatedRegister` | `string` | `src/mujoco-register.gen.d.ts` | Output declaration file |
| `moduleName` | `string` | `mujoco-react` | Module to augment |
| `disableLogging` | `boolean` | `false` | Disable plugin console output |
